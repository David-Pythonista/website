Searcher

Description of program:

Choose a sample pattern (txt file) and a target file (txt file). The program will count how many times occure the sample pattern in the target file.

In the first line, we have to import every necessary item. The Tkinter objects to create the interface of the program, the filedialog which ensure the communication between the operating system and our application. Beside buttons, we need input field (Entry) and tkinter variable to store data.

The init function creates the interface, stores the result in the self.output variable, and put it int the entry field. In this case the program use it, to show result for the user. Also create the lists, to store the data what was given.

This section contains the function behind the push buttons. Load the sample and target file. Both of them are tested by the „tester” function, are they empty or not. Empty files are not accepted.

Length of sample counts how many character have the sample file. This data will be used when the program try to find the sample pattern in the target file.

The start function will run the whole process. It does not accept empty files!.

Match finder will go through the lists which were made in the init section, and contain the given data.

If the application find a character what is mathes with the first character of the sample, stop goes through the file, and check the next elements (checked elements number is the same as the length of sample, that’s why calculated before).

It will counts the full matches and write it in the entry field.

This is the last - main - part of the program that will run the application. The "if" condition means that, if the program run directly then it will works, otherwiswe not.

The interface of the program. With the "Load sample button" the user can open a window to select a pattern what must be found int the target file, which could be scrolled with "Load target file" button.
